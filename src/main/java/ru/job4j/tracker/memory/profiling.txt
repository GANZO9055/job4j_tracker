Файл создан для описания анализа состояния памяти (в некоторых моментах состояния CPU)

Будут произведены три действия:
1. Создание заявок
2. Показ всех заявок
3. Удаление всех заявок

Анализ будем проводить по трем ситуациям:
1. При 100 заявок
2. При 10_000 заявок
3. При 100_000_000 заявок (в данном случае попытаемся выйти за пределы памяти)

Анализ будет производиться с помощью инструмента jconsole

Анализ при 100 заявок
При создании заявок было незначительное увеличение памяти (потребление CPU составляло 0.4%),
однако потребление памяти продолжало увеличиваться независимо от количества заявок.
При выводе всех заявок, изменений с памятью не наблюдалось (потребление CPU составляло 0.1%)
При удалении заявок, изменений с памятью не наблюдалось (потребление CPU составляло 0.2%)

Анализ при 10_000 заявок
При создании заявок произошел резкий скачек по памяти на 5 Мбайт (потребление CPU также составляло 0.4%)
При выводе всех заявок, произошел резкий скачек по памяти на 7 Мбайт (потребление CPU составляло 1.6%)
При удалении заявок, изменений с памятью не наблюдалось (потребление CPU составляло 0.2%),
чуть позже произошло незначительное увеличение пямяти

Анализ при 100_000_000 заявок
При создании заявок был достигнут предел по пямяти, который составлял 4.1 Гб (потребление CPU также составляло 64.9%),
после выпадало исключение OutOfMemoryError.
